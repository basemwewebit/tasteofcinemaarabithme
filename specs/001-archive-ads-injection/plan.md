## Summary

This feature implements automated ad injection across archive-type pages (`archive.php`, `category.php`, `author.php`) and the 404 error page. The core logic involves modifying the post loops in archive templates to inject ad blocks every 6 posts and adding dedicated ad slots to the 404 page, ensuring consistency with the site's overall design system.

## Technical Context

**Language/Version**: PHP 8.x  
**Primary Dependencies**: WordPress Core, Secure Custom Fields (SCF)  
**Storage**: WordPress Database (Options table via SCF)  
**Testing**: Manual responsive UI verification and ad slot validity checks  
**Target Platform**: Linux / WordPress  
**Project Type**: WordPress Theme  
**Performance Goals**: <50ms server-side overhead for ad injection logic  
**Constraints**: RTL layout (Arabic), responsive grid (1-3 columns)  
**Scale/Scope**: 4 primary template files, 1 configuration file, and theme options
